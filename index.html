悬案② 破案 · 上下文工程层 · 2.5′

越聊越笨,不是它累了——
是压缩把记忆吃了

  • 忘了半小时前定好的约定,用回默认做法

失忆樵夫,与背包经济学

MD = 背包里的字条该“知道”的:规范、约定、坑——只改变输出,不给能力
Tool = 手里的斧子该“做”的:可校验、可拦截——护栏只能装在斧子上
CLI = 山下五金铺人类五十年的现成工具,一把钥匙(Bash)全借走
MCP = 集市摊位标准跨家复用的接口——摊位后面,还是斧子
万物皆占背包(上下文窗口)，默认不进。

怎么配?一张判断表

层放什么代价
MD只需“被读到”的知识 → 越短越好按字数常驻背包,过量 = context rot
Tool高频、关键、要护栏的动作 → 才封 Toolschema 说明卡也占背包
CLI人类已有现成工具 → 别重造轮子无护栏 = 最大攻击面,沙箱防的就是它
MCP跨 agent / 跨团队标准化接入 → 才开不是第四种能力,是工具的供应链
底层结构 2×2：(知识 vs 能力)×(自有 vs 生态)。
