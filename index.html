悬案② 破案 · 上下文工程层 · 2.5′

越聊越笨,不是它累了——

控制流层 · Codex Goal

Codex Goal：
把“继续做”变成 harness 状态

模型负责什么

读当前上下文,做一轮推理,产出下一步文本或工具调用

Goal harness 负责什么

保存目标、空闲续跑、检查是否完成,卡住时把控制权交还人

它解决的是：没人继续追问时，任务还能按目标推进。

控制流层 · Claude Code Workflow

Workflow：
Claude Code 里的“Ultracode”

屏幕上看到的标签: (Ultracode - xhigh + workflows)
拆开看 ↓
xhigh: 更高推理预算,让模型先规划、拆解,再决定怎么行动
再加上 ↓
workflows: 把拆分、并发、合并、验证交给 harness 的编排脚本
所以 Ultracode 更像“高推理档位 + workflow 调度”,不是单纯换了一个更聪明的模型。

失忆樵夫,与背包经济学

MD = 背包里的字条该“知道”的:规范、约定、坑——只改变输出,不给能力
Tool = 手里的斧子该“做”的:可校验、可拦截——护栏只能装在斧子上
CLI = 山下五金铺人类五十年的现成工具,一把钥匙(Bash)全借走
MCP = 集市摊位标准跨家复用的接口——摊位后面,还是斧子
万物皆占背包(上下文窗口)，默认不进。
来源：CLAUDE.md / memory · Model Context Protocol · context rot

怎么配?一张判断表

层放什么代价
MD只需“被读到”的知识 → 越短越好按字数常驻背包,过量 = context rot
Tool高频、关键、要护栏的动作 → 才封 Toolschema 说明卡也占背包
CLI人类已有现成工具 → 别重造轮子无护栏 = 最大攻击面,沙箱防的就是它
MCP跨 agent / 跨团队标准化接入 → 才开不是第四种能力,是工具的供应链
底层结构 2×2：(知识 vs 能力)×(自有 vs 生态)。参考：Claude Code permissions · Tool design
